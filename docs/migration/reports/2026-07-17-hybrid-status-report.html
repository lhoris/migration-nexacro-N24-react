⚠ 아직 커밋되지 않음 — 전부 워킹 디렉토리 변경사항
한 줄 요약 — React Host Shell이 메인프레임(로고·GNB·LNB·Footer·홈 화면·언어/테마)을 완전히 소유하는 상태로 전환됐고, 그 안에서 메뉴 화면 1개(정렬·필터·찾기)가 React 네이티브로 100% 동작한다. 나머지 39개 메뉴는 지금도 진짜 Nexacro/Tomcat이 그대로 서빙하며, React 셸에서 클릭하면 실제 그 화면으로 풀 페이지 이동한다 — 목업이나 iframe이 아니다.

1 · 지금 실제로 떠 있는 것

로컬에서 게이트웨이(:3000) 하나로 두 스택이 실제로 붙어 있는 상태다.

React Host Shell
vite build → nginx :3000 "/"
  • GNB(로고/그룹메뉴/언어·테마/메가메뉴)
  • 홈 화면(원본 3섹션 이식)
  • LNB, Footer
  • 메뉴 화면 1개: 정렬·필터·찾기
⇄
Nexacro (레거시)
Tomcat 9 :8080 → nginx "/nexacro/**"
  • 메뉴 화면 39개 그대로 서빙
  • root.xfdl 해시 라우팅으로 딥링크 수신
  • 콜드 딥링크 · 맥 모바일 오탐 버그 수정됨

두 스택은 iframe으로 묶은 게 아니라 풀 페이지 네비게이션으로 오간다 — 세션/인증 통합은 아직 설계 전(§4 참고).

4 · 시연 전에 알아야 할 한계

!
세션·인증 통합 미착수 React 셸과 Nexacro는 지금 완전히 독립된 두 앱을 풀 네비게이션으로 오가는 것뿐이다. 로그인 상태·세션 공유 설계는 아직 손대지 않았다 — 고객이 "로그인하면 두 화면 다 유지되나?" 물으면 지금은 "아니오, 설계 예정"이 정답이다.
!
REST 계약 계층(ADR-003) 없음 게이트웨이의 /api/**는 지금 그냥 Tomcat으로 프록시만 걸어둔 자리 표시자다. React가 자체 API를 호출하는 화면은 아직 없다(정렬·필터·찾기도 프론트 목업 데이터로 동작).
!
target 전환 버튼은 로컬 시뮬레이션 미전환 화면(WorkArea)의 "target 전환 시뮬레이션" 버튼은 React 상태만 바꾸는 데모용이다. 실제로 그 화면이 React로 옮겨간 게 아니라는 점을 시연 중 명확히 짚어야 한다.
!
아직 커밋 전 이번 세션까지의 모든 변경은 워킹 디렉토리에만 있다. 데모 서버를 다시 띄우거나 다른 브랜치로 넘어가면 사라진다 — 데모 직전에 커밋 여부를 확인해야 한다.

5 · 시연 시나리오 제안

  1. 홈 화면 — React가 메인프레임을 소유한다는 것부터 보여주기
    롤링 배너 점 클릭으로 스텝 전환, 스크롤해서 레퍼런스 통계 카운트업 애니메이션 확인. 우측 상단 메가메뉴(▦) 열어서 전체 메뉴 구조 한눈에 보여주기.
  2. "그리드 → 정렬, 필터, 찾기" — 진짜로 전환된 화면 시연
    헤더 클릭 정렬, Ctrl/⌘+클릭 다중 정렬, 필터 아이콘, 찾기까지 실제 동작 시연. 화면 우상단 "원본 Nexacro 화면으로 이동" 버튼으로 원본과 나란히 비교.
  3. "그리드 → 페이징" — React가 아니라 진짜 Nexacro라는 것 강조
    클릭하면 풀 페이지 이동 — 주소창 URL이 /nexacro/launch.html#10300으로 바뀌는 걸 보여주며 "이건 흉내가 아니라 실제 레거시 엔진"이라고 설명.
  4. 아무 다른 메뉴 클릭 — 나머지 39개의 실제 상태
    "아직 전환 전이라 Nexacro가 그대로 서빙하지만, 딥링크 자체는 React 셸에서 진짜로 연결돼 있다"는 점을 짚기.
  5. 언어 EN 전환 + 테마 다크 전환
    셸 전체(GNB·LNB·홈·정렬화면)가 한번에 같이 바뀌는 것으로 "플랫폼 레벨 기능"이라는 점 강조.
